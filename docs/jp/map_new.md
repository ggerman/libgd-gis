## スタイル指定について（重要）

`GD::GIS::Map` は、**スタイルが関連付けられていない場合、レンダリングできません**。

`style` オブジェクトは、道路・水域・公園・ポイントなどの意味的レイヤーを、
色、線幅、塗りつぶし、描画順序といった視覚的属性へ変換する役割を担います。

そのため、以下が必須となります：

- `map.render` を呼び出す前に **必ずスタイルを設定する必要があります**
- スタイル未設定でレンダリングを行うと、実行時エラーが発生します
- これは仕様上の制約ではなく、**意図的な設計判断**です

例：

```ruby
map = GD::GIS::Map.new(
  bbox: PARIS,
  zoom: 13,
  basemap: :carto_light
)

map.style = GD::GIS::Style.load("default")

map.render
map.save("paris.png")
```

### なぜスタイル指定が必須なのか

libgd-gis は、暗黙的またはハードコードされた
デフォルトの視覚スタイルを提供しません。

スタイルを明示的に指定することで、以下が保証されます：

- レンダリング結果が常に決定的であること
- 視覚表現の意味が完全にユーザーの管理下にあること
- 色やレイヤーに関する隠れた前提が存在しないこと
- スタイル定義をバージョン管理・レビュー・再現可能に保てること

この仕様は、**タイルベース描画モード**および
**ビューポート描画モード**の両方に適用されます。
